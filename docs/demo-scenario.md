# 무중단 시연 시나리오 (US-22)

## 시연 환경

| 구성요소 | 주소 | 상태 확인 |
|---------|------|---------|
| 백엔드 | https://backend.pongchi.kro.kr | GET /health |
| 클라이언트 | Electron 앱 실행 | 로그인 화면 |
| ML 서버 | 192.168.0.3:8001 (LAN) | GET /health |

---

## 시연 순서

### 1단계 — 회원가입 / 로그인 (1분)

1. Electron 앱 실행
2. "회원가입" 클릭 → 이메일/비밀번호 입력 → 가입 완료
3. 로그인 → 메인 화면 진입

---

### 2단계 — 음성 녹음 & 추천 (2분)

1. 메인 화면에서 **"음성 녹음"** 버튼 클릭
2. 5초간 일상 발화 녹음 (예: "오늘 너무 피곤하고 힘들어")
3. 자동으로 `/recommend` 호출 → 추천 결과 화면으로 전환
4. 결과 확인:
   - **감정 차트**: Valence/Arousal 2D 매핑
   - **추천 곡 리스트**: 5곡 + 추천 이유 카드
   - **STT 변환 결과**: 인식된 텍스트

---

### 3단계 — 피드백 & 재생 (1분)

1. 추천 곡 미리듣기 재생 (▶ 버튼)
2. 마음에 드는 곡에 **좋아요(👍)** 클릭
3. 마음에 안 드는 곡에 **싫어요(👎)** 클릭

---

### 4단계 — 추천 이력 조회 (30초)

1. 메인 화면 → "이력" 탭 클릭
2. 이전 추천 세션 목록 확인

---

### 5단계 — 장애 내성 시연 (선택, 1분)

ML 서버를 잠시 종료하고 `/recommend` 재시도:
- **기대 결과**: neutral fallback VAD(0,0,0)으로 추천 정상 반환 (서비스 중단 없음)

---

## 비상 대응 체크리스트

| 문제 | 대응 |
|------|------|
| 백엔드 응답 없음 | `docker compose -f docker-compose.blue-green.yml restart backend-blue` |
| ML 서버 응답 없음 | 데스크탑에서 `serve.bat` 재실행 |
| 추천 결과 안 나옴 | 브라우저 devtools → Network 탭 → 400/500 에러 확인 |
| 녹음 안 됨 | 마이크 권한 확인 (OS 설정) |

---
